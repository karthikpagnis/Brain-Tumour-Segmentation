# API Documentation

## Base URL

```
http://localhost:8000
```

## Authentication

Currently no authentication required. In production, add API keys:

```
Authorization: Bearer <api_key>
```

## Endpoints

### 1. Health Check

**GET** `/api/health`

Check API and model status

**Response:**

```json

	"model_loaded": true,
	"device": "cuda"
}
```

### 3. Brain Tumor Segmentation (Main Endpoint)

**POST** `/api/predict`

Predict tumor segmentation from MRI volume

**Headers:**

```
Content-Type: multipart/form-data
```

**Parameters:**
| Name | Type | Required | Description |
|------|------|----------|-------------|
| file | File | Yes | NIfTI file (.nii, .nii.gz) |

**Request Example:**

```bash
curl -X POST \
  -F "file=@patient_scan.nii.gz" \
  http://localhost:8000/api/predict
```

**Response (Success - 200):**

```json
{
	"status": "success",
	"message": "Segmentation completed successfully",
	"processing_time": 3.45,
	"output_filename": "prediction_20250411_153022.nii.gz",
	"predictions_url": "/api/download/prediction_20250411_153022.nii.gz",
	"class_names": {
		"0": "Background",
		"1": "Necrotic Core",
		"2": "Peritumoral Edema",
		"3": "Enhancing Tumor"
	}
}
```

**Response (Error - 400):**

```json
{
	"detail": "Invalid file format. Expecting .nii or .nii.gz"
}
```

---

### 4. Download Prediction

**GET** `/api/download/{filename}`

Download segmentation result as NIfTI file

**Parameters:**
| Name | Type | Description |
|------|------|-------------|
| filename | string | Output filename from predict endpoint |

**Request:**

```bash
curl -O http://localhost:8000/api/download/prediction_20250411_153022.nii.gz
```

**Response:**

- Binary NIfTI file (with gzip compression)
- Use nibabel in Python to read:

```python
import nibabel as nib
pred = nib.load('prediction.nii.gz')
print(pred.get_fdata().shape)  # (155, 240, 240)
```

## Error Codes

| Code | Message             | Solution                            |
| ---- | ------------------- | ----------------------------------- |
| 200  | Success             | OK                                  |
| 400  | Invalid file format | Upload .nii or .nii.gz file         |
| 404  | File not found      | Check filename in download endpoint |
| 503  | Model not loaded    | Restart API server                  |
| 413  | File too large      | Max size: 100 MB                    |

## Usage Examples

### Python (requests)

```python
import requests

# Upload and predict
url = "http://localhost:8000/api/predict"
with open("mri_scan.nii.gz", "rb") as f:
    files = {"file": f}
    response = requests.post(url, files=files)

result = response.json()
print(f"Processing time: {result['processing_time']:.2f}s")

# Download result
download_url = "http://localhost:8000" + result['predictions_url']
pred = requests.get(download_url)
with open("prediction.nii.gz", "wb") as f:
    f.write(pred.content)
```

### Python (nibabel integration)

```python
import nibabel as nib
import numpy as np

# Load prediction
pred_nii = nib.load("prediction.nii.gz")
pred_array = pred_nii.get_fdata()  # (155, 240, 240)

# Extract tumor mask
tumor_mask = pred_array > 0

# Count voxels per class
classes = {
    0: "Background",
    1: "Necrotic Core",
    2: "Edema",
    3: "Enhancing"
}

for class_id in range(1, 4):
    count = np.sum(pred_array == class_id)
    print(f"{classes[class_id]}: {count} voxels")
```

## Rate Limiting

Current: No rate limiting (add in production)

Recommended for production:

- 10 requests/minute per IP
- 100 requests/day per API key
- Queue long-running tasks

---

## Data Privacy

- All files processed locally
- Results stored temporarily in `outputs/`
- No data sent to external servers
- Clean up old predictions regularly

---

## Performance Notes

- **Inference time**: 2-5 seconds per volume (GPU)
- **Memory**: ~4.5 GB per concurrent request
- **Batch processing**: Supported for multiple files
- **Concurrent requests**: Limited by GPU memory
